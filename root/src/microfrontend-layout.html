Como utils esta pensado mas para exportar funcionalidad que componentes visuales se tienen que hacer unos cambios en la configuracion de babel en webpack para que se pueda transpilar el codigo de jsx (react) a js

Sobre esta app

Esta app utiliza un layout que da single-spa esto es muy bueno cuando queremos tener cosas como navbars (que es una app que se mostrará siempre) como la que tenemos en esta página ademas de que manejamos el enrutamiento de una forma un poco mas visual que sin este layaout por ejemplo, esto que estas leyendo no esta en ninguna app pero como tengo el layout de single-spa puedo adornar con algunas cosas un poco mas estaticas la apariencia de mi página.

Dependencias compartidas

Esta app a su ves utiliza para las dependencias compartidas algo conocimo como importmap que no son mas que archivos estaticos los cuales importamos desde el ejs, se recomienda hacer importmaps de bibliotecas grandes como react, angular o vue, pero no de bibliotecas pequeñas ya que esto no nos traeria todos los beneficios que esperamos, es decir el beneficio obtenido practicamente ni se notara.

Cosas por hacer

Compartir dependencias por medio de Module Federation Plugin, en su página mencionan que se puede hacer pero es una caracteristica que aun no está muy estudiada y que Module Federation Plugin está en cambio por lo que no se profundiza en como hacerlo.

Recomendaciones

Se recomienda que cada microfrontend sea lo menos acoplado a los demas y que cuando se compartan componentes que sea por medio de parcels ya que estos puede adaptarse a cualqueir framework, por ejemplo si comparto un componente de react para utilizarlo en una app angular no seran compatibles, pero si comparto un componente como un parcel este si se puede mapear entre los frameworks, en esta app se abarcan ambos casos.

La información que se compartirá en varios microfrontends como tokens o API´S se espera que se comparta por medio de una app de utils ya que asi se maneja de una forma centralizada evitando duplicar código.

Lazy load

Si utilizamos los metodos de Lazy Load de los frameworks SPA pues ya lo tendriamos, sin embargo hay un metodo con los parcels que es por medio de loadRootComponent el cual retorna una promesa con el compoente a renderizar.
